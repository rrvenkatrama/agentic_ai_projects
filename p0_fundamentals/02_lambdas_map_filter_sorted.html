Lambdas — Anonymous Functions

A lambda is a tiny one-line function with no name. Two forms produce the exact same function:

def square(x):
    return x * x

square_lambda = lambda x: x * x

square(5)         # 25
square_lambda(5)  # 25

Syntax

lambda <args>: <single expression>

# Examples
lambda x: x * 2
lambda a, b: a + b
lambda name, prefix="Hello": f"{prefix}, {name}"
lambda x: x if x >= 0 else -x      # ternary OK
When to use lambda vs def
  • Use lambda when passing a tiny one-shot function as an argument: sorted(x, key=lambda t: t.price)
  • Use def when the function has a name worth referring to, has multiple lines, or is used in more than one place

Lambda Limitations

AllowedNot allowed
One expressionStatements (assign, loop, print, return)
Conditional expression (x if cond else y)Multi-line bodies
Default argumentsAnnotations easily
 
# ❌ SyntaxError — has multiple statements
bad = lambda x: print(x); return x

# ❌ SyntaxError — assignment is a statement
bad = lambda x: y = x * 2

# ✅ Conditional expression is fine
classify = lambda price: "expensive" if price > 200 else "cheap"

map(func, iterable)

Apply a function to every item in an iterable. Returns an iterator (lazy), not a list.

prices = [100, 200, 300]

doubled_iter = map(lambda x: x * 2, prices)
print(doubled_iter)         # <map object at 0x...>   ← iterator, not list
print(list(doubled_iter))   # [200, 400, 600]         ← consume to see

Map over multiple iterables

a = [1, 2, 3]
b = [10, 20, 30]
list(map(lambda x, y: x + y, a, b))   # [11, 22, 33]
⚠️ map iterators are exhausted after one pass
m = map(lambda x: x * 2, [1, 2, 3])
list(m)        # [2, 4, 6]
list(m)        # []  ← already consumed

filter(func, iterable)

Keep only items where func(item) returns truthy. Returns an iterator.

prices = [100, 250, 175, 300, 50]
expensive = list(filter(lambda x: x > 200, prices))
print(expensive)   # [250, 300]

filter(None, iterable) — drop falsy values

mixed = [1, 0, "hello", "", None, [1], []]
list(filter(None, mixed))   # [1, 'hello', [1]]

Removes anything Python considers falsy: 0, "", None, [], {}, False.

sorted(iterable, key=…, reverse=…)

Returns a NEW sorted list. Doesn't mutate the input.

nums = [3, 1, 4, 1, 5, 9, 2, 6]
sorted(nums)                  # [1, 1, 2, 3, 4, 5, 6, 9]
sorted(nums, reverse=True)    # [9, 6, 5, 4, 3, 2, 1, 1]

# Strings — alphabetical by default
sorted(["MSFT", "AAPL", "GOOGL"])    # ['AAPL', 'GOOGL', 'MSFT']

# Sort by length using key
sorted(["MSFT", "AAPL", "GOOGL"], key=len)    # ['MSFT', 'AAPL', 'GOOGL'] (4,4,5)

list.sort() vs sorted()

🚨 list.sort() returns None
tickers = ["MSFT", "AAPL"].sort()    # ❌ tickers is None!

# ✅ Either:
sorted_tickers = sorted(["MSFT", "AAPL"])
# Or:
tickers = ["MSFT", "AAPL"]
tickers.sort()     # mutate, then use tickers

Sorting Dicts and Objects

Sort list of tuples by element index

prices = [("AAPL", 150), ("GOOGL", 2800), ("MSFT", 300)]
sorted(prices, key=lambda x: x[1])
# [('AAPL', 150), ('MSFT', 300), ('GOOGL', 2800)]

# Same with operator.itemgetter (slightly faster, more idiomatic)
from operator import itemgetter
sorted(prices, key=itemgetter(1))

Sort list of dicts by key

portfolio = [
    {"ticker": "AAPL", "price": 150, "pe": 34},
    {"ticker": "GOOGL", "price": 2800, "pe": 28},
    {"ticker": "MSFT", "price": 300, "pe": 35},
]

sorted(portfolio, key=lambda x: x["pe"])
# Sorted by PE ascending

Sort list of objects by attribute

from operator import attrgetter

class Stock:
    def __init__(self, ticker, price): ...

stocks = [Stock("AAPL", 150), Stock("GOOGL", 2800)]

sorted(stocks, key=lambda s: s.price)        # via lambda
sorted(stocks, key=attrgetter("price"))      # via attrgetter (cleaner)

Multi-Key Sorting

Return a tuple from the key function. Python compares tuples lexicographically — first element, then second, etc.

# Sort by PE ascending, then by price ascending (within same PE)
sorted(portfolio, key=lambda x: (x["pe"], x["price"]))

Mixed asc/desc — negate numeric fields

# Sort by PE ascending, then price DESCENDING
sorted(portfolio, key=lambda x: (x["pe"], -x["price"]))

# For non-numeric descending, sort twice (Python sort is stable):
result = sorted(portfolio, key=lambda x: x["price"], reverse=True)
result = sorted(result, key=lambda x: x["pe"])
# Final: PE asc, with price desc within each PE
Stable sort matters
Python's sorted is stable — items with equal keys keep their original order. That's why sorting twice (last-key-first) gives correct multi-key results when negation isn't possible.

reduce() — less common

from functools import reduce

nums = [1, 2, 3, 4, 5]

reduce(lambda acc, x: acc + x, nums)         # 15  (sum)
reduce(lambda acc, x: acc * x, nums)         # 120 (product)
reduce(lambda acc, x: acc + x, nums, 100)    # 115 (with initial value)
Use builtins instead when possible
sum(nums), max(nums), min(nums), any(nums), all(nums) are clearer than reduce for those operations.

Common Agentic Code Patterns

Filter LLM response for tool_use blocks

tool_calls = [b for b in response.content if b.type == "tool_use"]

Sort search results by relevance

top = sorted(search_results, key=lambda x: x["score"], reverse=True)[:5]

Sort tools by name for consistent display

from operator import attrgetter
sorted_tools = sorted(mcp_tools, key=attrgetter("name"))

Convert MCP tool list to Claude format

claude_tools = [
    {"name": t.name, "description": t.description, "input_schema": t.inputSchema}
    for t in tools_result.tools
]

Sort messages by timestamp, prioritizing system messages

# role priority: system=0, user=1, assistant=2 (system first within same time)
priority = {"system": 0, "user": 1, "assistant": 2}
sorted_messages = sorted(messages, key=lambda m: (m["timestamp"], priority[m["role"]]))
