Complete execution sequence for crew.py / agents.py / tools.py  ·  2026-04-26

Agent = System Prompt, Task = User Prompt

The clearest mental model for CrewAI: an Agent is what you'd put in a system prompt, a Task is what you'd put in the user message.

Raw LLM callCrewAI equivalent
system promptAgent — role + goal + backstory
user messageTask — description + expected_output
tool definitionsAgent.tools=[]
tool result appended to messagesTask.context=[prior_task]
LLM responseTask output — stored, passed to next task

If you were doing this manually without CrewAI, one agent + task would look like:

client.messages.create(
    model="claude-opus-4-7",
    system="You are a Senior Code Reviewer. Goal: find bugs...",  # ← Agent
    messages=[{
        "role": "user",
        "content": "Review this PR diff: [diff text here]"           # ← Task
    }]
)

CrewAI automates this pattern — builds the system prompt from the Agent, the user message from the Task description, injects prior task outputs via context=, and handles the tool call loop.

Import Chain — What Runs at Startup

Before run_review() is ever called, three files execute in sequence:

python crew.py
│
▼ line 4: from agents import code_fetcher, ...
agents.py runs top to bottom
   line 3: from tools import fetch_pr_details
     │
     ▼
   tools.py runs top to bottom
      → @tool decorator runs → fetch_pr_details becomes Tool object
     ▼ back in agents.py
   → Agent(...) called 3 times → code_fetcher, code_reviewer, tech_writer created
▼ back in crew.py
Task(...) called 3 times
   → fetch_task, review_task, summary_task created
   → {pr_url} placeholder NOT yet filled — just stored as template
▼
All objects ready in memory. Zero LLM calls so far.

Importing Objects — Not Just Functions

import works for anything Python can assign to a variable — not just functions.

from agents import code_fetcher    # an Agent object
from tools import fetch_pr_details  # a Tool object
from math import pi                 # a float (3.14159...)
from config import MAX_RETRIES     # an integer
Module caching: Python runs agents.py exactly once, then caches it in sys.modules. If multiple files import from agents.py, the module runs only on the first import. All importers share the same three Agent objects.

Step 2 — Crew assembled

The agents= list is just a declaration of who's on the team — order doesn't matter here. The tasks= list determines execution order. Still no LLM calls.

Step 3 — kickoff() + template substitution

LINE 38 
result = crew.kickoff(inputs={"pr_url": pr_url})

Two things happen immediately:

Template substitution: CrewAI scans all task descriptions for {variable} placeholders and fills them from inputs=:

# Before kickoff:
"Fetch the full details for this GitHub PR: {pr_url}"

# After substitution:
"Fetch the full details for this GitHub PR: https://.../pull/1"

Sequential execution begins — tasks run in the order of the tasks= list.

Step 4 — fetch_task runs (CodeFetcher agent)

TASK 1

CrewAI sends this to anthropic/claude-opus-4-7:

# System (from Agent):
You are a PR Data Retrieval Specialist.
Goal: Fetch the complete diff and metadata for a GitHub pull request.
Backstory: You are an expert at retrieving data from GitHub...
Tools available: fetch_pr_details — Fetches the title, description,
  changed files and diff. Input: a full GitHub PR URL.

# User (from Task description):
Fetch the full details and diff for this GitHub PR:
https://github.com/rrvenkatrama/agentic_ai_projects/pull/1
Expected output: The PR title, description, all changed files and diffs.

LLM responds (ReAct loop):

Thought: I need to fetch the PR details using the fetch_pr_details tool.
Action: fetch_pr_details
Action Input: {"pr_url": "https://.../pull/1"}

CrewAI calls fetch_pr_details.run(pr_url) → tools.py executes:

requests.get("https://api.github.com/repos/.../pulls/1")        # PR metadata
requests.get("https://api.github.com/repos/.../pulls/1/files")   # diff

Result returned to LLM as Observation. LLM responds with Final Answer. fetch_task output stored by CrewAI.

Step 7 — result returned

LINE 39 
return result.raw   # always the LAST task's output as a plain string

result is CrewAI's result object. result.raw is the final task's output as a plain string. result.pydantic would give a structured object if output_pydantic= was set on the last task.
